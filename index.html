    • Must be Equal to (===)
      • '8'=== 8 returns false
        • As one is a string and the other is a number
      • '8'==='8' returns true
        • As both is a string
    • Must be Not Equal to (!==)
      • 8' !== 8 returns true
        • As one is a string and the other is a number
      • '8' !== '8' returns false
        • As both is a string
    • Is Equal to (==)
      • 'Hello' == 'Goodbye' returns false
        • As the string content is different
      • 5 == '5' returns true
        • As the values is the same even though the type is different
    • Not Equal to (!=)
      • 'Hello' != 'Goodbye' returns true
        • As the string content is different
      • 'Hello' != 'Hello' returns false
        • As the values is the same even though the type is different
    • Greater than (>)
      • 8 > 7 returns true
      • 7 > 8 returns false
    • Lesser than (<)
      • 8 < 7 returns false
      • 7 < 8 returns true
    • Greater than or equal to (>=)
      • 8 >= 7 returns true
      • 7 >= 8 returns false
      • 8 >= 8 returns true
    • Lesser than or equal to (<=)
      • 8 <= 7 returns false
      • 7 <= 8 returns ture
      • 8 >= 8 returns true
    • And (&&)
      • ((2 < 5) && (3 >= 2)) returns true
      • As 2 is less than 5 and 3 is more than 2
      • Both must return true
    • Or (||)
      • ((2 < 5) || (3 >= 2)) returns true
      • Either one must return true in order to return true at the end
    • Not (!)

Week 12

API
  • How Requst is Form
    • API Link
    • Methods
      • GET: Retrive information from the api
      • POST: Send the new data into api
      • PATCH/PUT: Update existing data in api
      • DELETE: Delete a data from api
    • Header: Additional details provided for communication between client and server
    • Data
    Example
    • var settings = { "url": "https://api.data.gov.sg/v1/environment/p si?date=2020-12-01", "method": "GET",};
    Status Code
    • 200
      • Message: OK
      • Description: The data was received and the operations was performed
    • 201
      • Message: Created
      • Description: The data was received and a new resource was created. The response needs to return the data in the payload
    • 204
      • Message: No content
      • Description: Operations was successful but no data was returned in the response body. Useful when deleting a data
    • 400
      • Message: Bad Request
      • Description: Server cannot accept the request because something went wrong on client side or the request that is send
    • 403
      • Message: Forbidden
      • Description: Client is not allowed to access
    • 404
      • Message: Not Found
      • Description: Computer not able to find the resource
    • 405
      • Message: Method Not allowed
      • Description: when sending delete request to server that doesn't support that method
